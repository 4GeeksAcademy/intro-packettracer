Setting up the server

A server is a device on our network that, as its name suggests, provides services to other devices. In this case, the service we will configure is DHCP, which is responsible for automatically assigning IP addresses to all devices that connect to the network. This will save us from having to manually configure IP addresses on all devices one by one.

On "Server 0 "

  1. Click on the device icon to open its configuration window.
  2. Turn on the server by clicking the switch; the LED will turn green.
  3. Go to the "Config" tab. You may have to wait a short time while the device boots up, just like in real life.
  4. In the left panel, select the "FastEthernet0" interface.
  5. Within the interface, activate it with the "Port Status" checkbox.
  6. You will also need to configure an IP address and subnet mask:
    • IPV4 Address: 192.168.0.10
    • Subnet Mask: 255.255.255.0
    We will use a different IP address, but within the same range. The calculations for these IP ranges are beyond the scope of this course and will always be provided to you.
  7. In the "Settings" tab, you must specify a default gateway. For this, we will use the IP address of the router that represents our internet connection: 192.168.0.1.
  8. With this, the interface is ready to connect to the network. On this device, changes are saved automatically.

Configurando el servidor

El servidor es un dispositivo de nuestra red, que como su nombre lo indica, presta servicios a los demás dispositivos. En este caso el servicio que configuraremos es el DHCP, que se encarga de asignar direcciones IP automáticamente a todos los dispositivos que se conecten a la red. Con esto evitaremos tener que configurar direcciones IP manualmente a todos los dispositivos uno por uno.

En el "Server 0"

  1. Haciendo click en el icono del dispositivo se abre la ventana de configuración de este
  2. Enciende el servidor haciendo click en el interruptor, verás como el led cambia a verde
  3. Luego deberías ir a la pestaña "Config". Puede que tengas que esperar un breve tiempo mientras el equipo enciende, justo como en la vida real
  4. En el panel izquierdo, selecciona la interfaz "FastEthernet0"
  5. Dentro de la interfaz, activala con la casilla de verificación de "Port Status"
  6. Ademas también tendrás que configurar una dirección IP y una mascara de subred
    • IPV4 Address: 192.168.0.10
    • Subnet Mask: 255.255.255.0
    Colocaremos una dirección IP distinta, pero dentro del mismo rango. Los cálculos de estos rangos de IP quedan fuera del alcance del curso, y siempre te serán proporcionados.
  7. En la pestaña "Settings" debes especificar una puerta de enlace por defecto "Default Gateway", para ello utilizaremos la dirección IP del router que representa nuestra conexión a internet: 192.168.0.1
  8. Con esto queda lista la interfaz para conectarse a la red. En este dispositivo los cambios quedan guardados automáticamente.

Connecting the server to the switch

  1. To use the auto-connect feature, we will select the orange lightning bolt icon in the "Connections" section.

Conectando el servidor al switch

  1. Para esto utilizaremos la función de conexión automática, para esto seleccionaremos el icono del rayo naranja en la sección "Connections"
  1. When the cursor changes, we can click on the server and then on the switch to create the link. This tool automatically determines the correct link type and selects a compatible interface.
  1. Cuando el cursor cambie, podremos hacer click en el servidor y luego en el switch para hacer el enlace. Esta herramienta determina automáticamente el tipo de enlace correcto y selecciona una interfaz compatible
